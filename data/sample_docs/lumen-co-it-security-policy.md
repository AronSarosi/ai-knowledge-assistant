# IT Security Policy (2026)

*This is a sample document, part of a small set the assistant searches. The company
and the details below are fictional.*

## Why this matters

Every the company employee is responsible for keeping company and client data safe.
Most security problems are not clever attacks; they are ordinary mistakes - a reused
password, a laptop left on a train, a convincing email clicked in a hurry. This
policy is written so that doing the safe thing is also the easy thing. If a rule
here ever makes your job genuinely harder, tell IT and we will find a better way
rather than have you work around it.

## Passwords and accounts

Use the company password manager for all work accounts, and let it generate a unique
password for each one. Never reuse a password across services, because a leak on one
site then puts every account at risk. Work passwords must be at least 14 characters
long; the password manager handles this for you so you never have to invent or
remember them.

Multi-factor authentication (MFA) is mandatory on every account that supports it,
including email, the code repositories, the design tools, and the client database.
Treat your MFA codes like keys to the building. Never share a password or an MFA
code with anyone, including IT - we will never ask you for them, and any message
that does is an attempt to trick you.

## Devices

Do any work that involves client data only on a the company-issued laptop, never on
a personal device. Keep your laptop locked when you step away from it, even at home,
and make sure disk encryption is on - it is enabled by default on every machine we
issue, and you should not turn it off.

Keep your operating system and applications up to date; updates usually contain the
security fixes that matter most, so install them promptly when prompted rather than
postponing them for weeks. If your device is behaving strangely - unexpected
pop-ups, programs you did not install, sudden slowness - stop using it for sensitive
work and contact IT.

## Handling client and company data

Only access the client and company data you actually need for your work. Do not copy
client data onto personal devices, personal cloud storage, or USB drives. When you
need to share a file, use the company tools and share it with named people, not a
public or "anyone with the link" setting.

Client data is kept only as long as we need it and is deleted within 30 days of a
project ending or a client closing their account. If you are ever asked to handle
data in a way that feels wrong, pause and check with IT or a director before doing
it.

## Email and phishing

Be cautious with links and attachments, especially when a message is unexpected. The
warning signs are consistent: the message creates urgency or fear, it asks you to
log in or confirm credentials through a link, it comes from a slightly-wrong address,
or it asks you to break a normal process "just this once". When something feels off,
trust that feeling.

Do not click the link to "check". Instead, open the service directly from your
bookmarks or the password manager, or contact the supposed sender through a number
or address you already know. If you are not sure, forwarding it to IT to check costs
nothing.

## Reporting a security incident

If you think something has gone wrong - a phishing email you clicked, a lost or
stolen laptop, a suspicious login alert, or a file shared with the wrong person -
report it straight away. Email security@lumen.example with what happened and when,
and for anything urgent also message the #security channel so it is seen
immediately.

Report first and worry later about whether it really "counts". We would far rather
hear about ten false alarms than miss one real incident, and the sooner we know, the
more we can limit any damage. You will never be blamed for reporting something in
good faith, even if it turns out the problem was your own mistake.
